## 종별 분석

### pH
- HNO₃에만 민감. HNO₃/N₂O₅=2.0에서 pH=3.65 (실험 3.61 근접).
- HONO, H₂O₂: 영향 없음.
- 메커니즘: HNO₃ → 강산, 직접 H⁺ 공급.

### NO₃⁻
- Baseline에서 이미 101µM (실험 63의 1.6배 과다).
- HNO₃ 추가 시 악화 (222µM까지). HNO₃ 자체가 NO₃⁻ source.
- HONO, H₂O₂: 변화 없음.
- **어떤 gas input으로도 줄일 수 없음.** N₂O₅ MT가 지배 (R98, 89%).

### NO₂⁻
- 모든 케이스에서 0.001~0.009µM (실험 3µM의 300~3000배 부족).
- HONO/NO₂=1.0에서도 0.004µM.
- **gas input sweep으로 해결 불가.** 소비 반응(R92: NO₃+NO₂⁻, k=1.2e9; R32: O₃+NO₂⁻, k=5e5; R77: OH+NO₂⁻, k=1e9)이 지배.

### H₂O₂
- H₂O₂ gas input에만 반응. 선형 비례.
- **H₂O₂/O₃=0.03에서 13.0µM (실험 11µM 근접).**
- HONO, HNO₃: 영향 없음.

### O₃
- H₂O₂ 증가 → O₃ 감소 (502→215nM). H₂O₂ → OH 생성 → O₃ 소비 촉진.
- HONO: 약간 증가 (radical chain 강화).
- HNO₃: 변화 없음.

## 실험 matching 전략

### 조절 가능 (자유 파라미터 3개)
1. **H₂O₂/O₃ 비율** → H₂O₂ 농도 결정. ~0.03에서 실험 근접.
2. **HNO₃/N₂O₅ 비율** → pH 결정. ~2.0에서 실험 근접.
3. **δ_gas** → NO₃⁻ 결정 (전체 MT flux 스케일링).

### 독립성
- H₂O₂ 비율과 HNO₃ 비율은 서로 간섭 없음 (독립적으로 H₂O₂, pH 조절).
- δ_gas는 모든 종의 MT flux에 영향 → pH/H₂O₂도 변함 → 3개 동시 조정 필요.

### 해결 불가 (현재 모델 한계)
- **NO₂⁻**: gas input으로 불가. reaction set 또는 rate constant 검토 필요.

## HONO/NO₂ × O₃ MT scaling 2D sweep (2026-04-14)

### 조건
- BC: gas_alpha, 종별 α_b, δ_gas=10mm
- 고정: HNO₃/N₂O₅=0.83, H₂O₂/O₃=0.03

### 핵심 발견

1. **NO₂⁻ = 2.89µM (★ case)**: HONO/NO₂=10, O₃×0.3에서 실험(3.0)에 근접.
   - pH=3.62, H₂O₂=15.3도 합리적 범위.
   - 단, NO₃⁻=241µM (실험 63의 3.8배, 악화).

2. **NO₂⁻와 O₃는 강한 반비례**: O₃가 줄어야 R32(O₃+NO₂⁻) 소비가 약해져 NO₂⁻ 생존.
   - O₃×1.0: NO₂⁻ ~0.005~0.05 (O₃ barrier 완전)
   - O₃×0.1: NO₂⁻ ~0.008~25 (barrier 약화, HONO 비례)

3. **Trade-off**: NO₂⁻↑ → NO₃⁻↑ (HONO 증가 → HNO₃ 대응 pH 하락 → acid-catalyzed NO₃⁻ 생성)

4. **HONO/NO₂=10은 비물리적**: 문헌 범위 0.1~1 대비 10배 과대. 모델의 O₃ barrier(1D diffusion 한계)를 보상하기 위해 필요.

5. **Stiffness 문제**: HONO=10, O₃×0.2 case에서 3692초(61분) 소요. 고 HONO + 저 O₃ 조합에서 radical chain transition이 stiff.

### 결론

- pH, NO₂⁻, H₂O₂를 동시에 실험 근사 가능 (★ case).
- **NO₃⁻ 과다는 해결 불가** — N₂O₅ MT가 지배하며 HONO 추가가 악화시킴.
- NO₂⁻ 부족의 근본 원인은 O₃ barrier (1D no-convection 한계).
- HONO 과대 설정은 대류 부재를 보상하는 effective parameter로 해석 가능.

## NO₂⁻ R92/R32 Knockout Test (2026-04-14)

### 조건
- BC: gas_alpha, 종별 α_b, δ_gas=10mm
- Gas: HONO/NO₂=0.33, HNO₃/N₂O₅=0.83, H₂O₂/O₃=0.03
- 스크립트: Figures/test/sweep/test_no2m_sensitivity.py

### 결과

| Case | NO₂⁻ (µM) | O₃ (nM) | pH |
|------|----------|---------|-----|
| Baseline | 0.004 | 302 | 3.81 |
| R92 × 0.1 | 0.005 | 266 | 3.81 |
| R92 OFF | 0.005 | 261 | 3.81 |
| **R32 OFF** | **5.4** | **4207** | **3.82** |
| R92+R32 OFF | 5.4 | 4195 | 3.82 |
| **실험** | **3.58** | **–** | **3.61** |

### 핵심 발견
- **R92 (NO₃+NO₂⁻, k=1.2e9)**: OFF해도 NO₂⁻ 변화 없음. 주 소비원 아님.
- **R32 (O₃+NO₂⁻, k=5e5)**: OFF → NO₂⁻=5.4µM (실험 3.58에 근접). **진짜 소비원.**
- R32 OFF 시 O₃가 14배 축적 (302→4207nM) — O₃가 더 이상 NO₂⁻에 의해 소비 안 됨.

### O₃ Barrier 메커니즘 (1D no-convection 한계)
```
표면(z=0): O₃=0.6µM, HONO 유입 → NO₂⁻ 생성 → R32 즉시 소비
  NO₂⁻ lifetime at surface = 1/(5e5 × 6e-7) ≈ 3초

Bulk(z>>0): O₃=7nM → NO₂⁻ lifetime ≈ 5분 → 축적 가능
  하지만 NO₂⁻가 surface O₃ barrier를 통과 못 함
```
- 실험에서 NO₂⁻=3.58µM 유지 가능한 이유: 대류/자연대류로 O₃ 분산, 공간 분리.
- 1D diffusion-only 모델에서는 모든 종이 같은 표면 셀로 유입 → co-location → 상호 소비.
- **코드 버그/rate constant 문제가 아닌 모델 구조적 한계.**
